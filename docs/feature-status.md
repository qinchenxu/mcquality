# 功能状态

本文档用于记录当前仓库中已经落地的功能、部分完成的功能，以及仍未开始或尚未接通的功能。

更新原则：

- “已完成”指代码已经接入主流程，并能在当前版本中直接使用。
- “部分完成”指已经有数据结构、界面骨架或占位实现，但还没有完整闭环。
- “未完成”指仍停留在规划或设计阶段。

## 已完成

### 核心品质系统

- 合成产物会自动生成装备品质。
- 战利品掉落会自动生成装备品质。
- 品质数据会写入物品的 CUSTOM_DATA。
- 品质会按固定倍率缩放装备属性。
- tooltip 会显示品质与倍率摘要。

### 客户端详情面板基础能力

- 主手持有受支持武器时，可按 I 键打开装备详情面板。
- 详情面板已经显式关闭 Minecraft 默认背景模糊。
- 详情面板支持基础的双栏布局。
- 详情面板支持自动换行。
- 详情面板支持滚轮滚动。
- 详情面板支持方向键和翻页键滚动。
- 详情面板支持基础滚动条显示。

### 详情面板已接入的真实数据

- 可显示当前装备名称。
- 可显示当前品质稀有度。
- 可显示装备类型。
- 可显示当前品质倍率。
- 可显示当前物品上已经存在的属性修正列表。

### 第一版技能与被动数据

- 代码中已经定义主动技能枚举：弧光斩、守御脉冲、震荡爆发。
- 代码中已经定义被动效果枚举：稳固锋刃、迅击节奏、巨力握持、破势压制。
- 新生成的高品质武器会按当前品质映射写入主动技能 ID。
- 新生成的武器会按当前品质映射写入被动效果 ID。
- 详情面板会读取并展示主动技能名称、说明、触发方式、冷却时间和粒子风格文本。
- 详情面板会读取并展示被动效果名称、说明和数值强度。

## 部分完成

- 已完成：技能枚举、基础文本、面板展示、按品质映射写入数据。
- 未完成：技能真实释放逻辑。
- 未完成：右键触发后的服务端判定。
- 未完成：冷却的真实执行与同步。
- 未完成：技能对实体或战斗结果的实际影响。

### 被动效果系统

- 已完成：被动效果枚举、文本、面板展示、按品质映射写入数据。
- 未完成：被动效果对属性或战斗流程的真实结算。
- 未完成：被动效果的触发条件、持续时间、内置冷却等机制。

### 详情面板 UI

- 已完成：基础面板、标题栏、内容区、自动换行、滚动。
- 已完成：解决默认背景模糊问题。
- 未完成：更成熟的视觉风格与最终版布局。
- 未完成：滚动条拖拽。
- 未完成：分页、分类切换或选项卡。
- 未完成：不同分辨率下的专门适配验证。

### 粒子系统

- 未完成：技能释放时的真实粒子效果。

### 网络层

- 已完成：网络注册入口骨架。
- 未完成：主动技能消息定义。

- 未完成：冷却或状态同步消息。

## 未完成

### 词条系统重构

- 还没有把当前“单一品质倍率”重构成“稀有度 + 多词条实例”。
- 还没有 affix 数据结构。
- 还没有 affix 资源 JSON。
- 还没有按武器、工具、护甲分池抽取词条。
- 还没有一位小数随机词条值生成。
- 还没有冲突组、词条权重、词条品质等机制。

### 技能真实触发链路

- 还没有右键技能释放入口。
- 还没有主动技能执行模板，例如范围斩击、护盾脉冲或冲击波。
- 还没有把技能数据真正用于战斗逻辑。

### 被动效果真实结算

- 还没有把被动效果接入属性聚合。
- 还没有把被动效果接入事件触发或战斗判定。

### 粒子与表现层

- 还没有真实粒子 Provider。
- 还没有粒子 JSON 和贴图资源。
- 还没有技能风格到粒子表现的真实映射。

### 数据驱动资源

- 还没有技能定义 JSON。
- 还没有词条定义 JSON。
- 还没有基于资源文件的加载与注册逻辑。

### 兼容与验证

- 还没有旧版仅含 quality 标签装备的迁移逻辑。
- 还没有系统化的单人验证清单落地。
- 还没有多人环境验证。
- 还没有完整的平衡性验证。

## 当前结论

当前版本已经不再只是“单一品质倍率 + tooltip”这一层，已经进入“有详情面板、有技能/被动数据字段、有客户端展示基础设施”的阶段。

但现阶段仍属于“结构已经铺开，玩法闭环尚未完成”的状态。最关键的缺口仍然是：

1. 真正的词条系统还没开始落地。
2. 技能和被动目前主要是展示数据，还没有真实行为。
3. 粒子和网络只有骨架，没有形成实际表现闭环。

## 推荐的下一步

1. 先把右键主动技能的真实执行链路接起来。
2. 再把当前“按品质映射技能/被动”升级成真正的词条实例结构。
3. 最后补粒子同步和数据驱动资源，形成第一版可玩的闭环。
